# 小雪工作台 — 功能规格文档

> 交付给设计模型使用。本文档描述每个面板的功能、数据源、交互行为。
> 设计模型只负责视觉（HTML/CSS），数据占位用假数据，留 `@data` 标记供后端对接。

---

## 一、设计方向

```
项目：小雪工作台
美学：赛博实用主义 (Cyberpunk Utilitarian)

使用者：钧钧，独自在电脑前
场景：赛前分析 / 赛后复盘 / TK知识维护
情绪：专注、判断、深度思考

核心感觉：2077 义体医生操作台 — 不是霓虹灯牌，是工具。
          暗底、信息密集不乱、数据像在呼吸、每个像素在干活。

设计 DNA：
  ✅ 暗色为主（非纯黑，偏深灰绿/深蓝灰）
  ✅ 等宽或半等宽字体做数据标签（终端感）
  ✅ 色彩克制：一个信号色（琥珀/橙红/青绿择一）
  ✅ 分隔线 > 卡片（用线条切区域，不用圆角卡片）
  ✅ 编辑态有激活反馈（hover 细线边框，focus 像终端光标）
  ✅ 交互硬切或快速淡入（无弹跳/弹性动画）

  ✕ 毛玻璃、渐变、发光阴影
  ✕ 圆角大卡片套小卡片
  ✕ Inter/Roboto/Open Sans
  ✕ 紫色+青色 AI 标配配色
```

## 三、面板详细规格

### 3.1 顶部栏

```
┌─────────────────────────────────────────────────────────┐
│  ◈ 小雪工作台          [AL ▾]  [所有选手 ▾]    v0.1      │
└─────────────────────────────────────────────────────────┘
```

**`@data: team-selector`**
- 下拉选择队伍，24 队（LPL 12 + LCK 12），按赛区分组
- 切换时三栏联动刷新
- 数据源：`GET /api/teams` → `[{short_name, name, region}]`

**`@data: player-selector`**
- 默认"所有选手"，切换队伍后更新该队选手列表
- 选具体选手后 TK 面板过滤为该选手的 TK
- 数据源：`GET /api/players?team=AL` → `[{name, role}]`

```

**字段清单**（对应 `team_3d_data` 表）：

| 字段 | 类型 | 可编辑 | 说明 |
|------|------|--------|------|
| `dim_1_name` / `dim_1_value` | 文本 | ✅ | 优势局习惯（A/B/C型 + 描述） |
| `dim_2_name` / `dim_2_value` | 文本 | ✅ | 劣势局习惯 |
| `dim_3_name` / `dim_3_value` | 文本 | ✅ | 胜负手位置 |
| `notes` | 多行文本 | ✅ | 战术笔记、阵容信息、分析师评论 |
| `region` | 文本 | ❌ 只读 | LPL / LCK |
| `updated_at` | 时间戳 | ❌ 自动 | 最后修改时间 |

**`@data: team-3d-panel`**

交互行为：
- 点击任一字段 → 进入编辑态（边框变为信号色，出现光标）
- 编辑态下按 Escape → 取消
- 点「保存修改」→ `PUT /api/team/AL` → 写回数据库 → 显示"已保存"提示（2秒后消失）
- 切换队伍时若有未保存修改 → 弹确认框
- 底部显示 `最后更新: 2026-06-08 10:43`

```

**`@data: tk-panel`**

数据源：`GET /api/tk/search?q=&team=AL&player=&tag=&limit=20`
→ 调 knowledge-rag API (localhost:8768)

CRUD 操作：

| 操作 | 触发 | 后端 |
|------|------|------|
| 搜索 | 输入关键词 + 回车/点击搜索 | `GET /api/tk/search` |
| 筛选 | 点击筛选标签 | 同上，加 filter 参数 |
| 编辑 | 点 ✎ → 内联编辑 → 确认 | `PUT /api/tk/{id}` |
| 删除 | 点 ✕ → 确认弹窗 | `DELETE /api/tk/{id}` |
| 新增 | 填写内容+标签 → 保存 | `POST /api/tk` |

交互行为：
- 搜索框输入时 300ms 防抖自动搜索
- TK 列表滚动到底自动加载更多（无限滚动）
- 编辑/新增后列表自动刷新
- 关联概念用 `[[概念名]]` 语法，渲染时显示为可点击链接
- 点击概念 → 右侧画像面板高亮相关条目

## 五、状态管理

**页面对状态的依赖**：
```
当前选中队伍 ← 顶部选择器
当前选中选手 ← 顶部选择器（可空）
TK 搜索关键词 ← TK 面板搜索框
TK 筛选标签 ← TK 面板筛选按钮
画像折叠状态 ← 右侧面板标题栏
三维编辑脏状态 ← 左侧面板是否有未保存修改
```

联动规则：
- 队伍变化 → 清空选手选择 → 刷新全部三栏
- 选手变化 → 刷新 TK 面板（过滤）+ 画像面板（高亮）
- TK 搜索/筛选 → 仅刷新 TK 列表

## 六、给设计模型的注意事项

1. **所有数据区域用假数据占位**，标注 `@data: xxx` 注释即可。我会替换成真实 API 调用。
2. **CSS 变量**集中定义主题色、字体、间距，方便我后续调整。
3. **编辑态样式**需要明确的视觉差异（不同于只读态），建议用边框颜色变化 + 微弱背景色偏移。
4. **不需要登录/用户系统**，单人使用。
5. **不需要移动端适配**，1920px 基准。
6. **暗色主题唯一**，不需要亮色切换。
7. **入口文件命名为 `index.html`**。
