## 前端架构（main.js）

### 状态管理

只用了一个全局对象 `state`：

```js
const state = {
  team: null,        // 当前选中的队伍（short_name）
  teams: [],         // 所有队伍列表
  _3d: null,         // 当前队伍的三维数据
  dirty: false,      // 三维数据是否有未保存的修改
  tradeGame: 'lol',  // 当前交易记录的游戏筛选
  trades: [],        // 当前加载的交易记录
  _tkResults: [],    // 当前 TK 搜索结果
};
```

### API 封装

```js
const API = (path, opts) => fetch('/api' + path, opts).then(r => {
  if (!r.ok) throw new Error(r.status);
  return r.json();
});
```

所有 API 请求通过 `/api/*` 前缀，在 Vite dev 模式下代理到 8880（生产直接同域）。

### 功能模块与数据流

#### 1. 队伍选择

- 初始化：`init()` 从 `/api/teams` 拉取全部队伍，按 LPL→LCK→INTL 排序
- 下拉菜单：`toggleTeamDropdown()` 渲染 LPL/LCK 分组列表
- 选择后：`selectTeam(code)` → 并行拉取 profile / wiki / 3D 数据
  - 优先从 wiki 目录（`/home/ubuntu/workspace/knowledge/wiki/小雪电竞/实体画像/队伍/`）加载
  - 回退到 SKILL.md（`~/.hermes/skills/{team_lower}-team-profile/SKILL.md`）
  - 自动搜索 TK（以队伍 code 为关键词）

#### 2. 命令解析（命令行驱动）

`runCommand()` 解析输入框文本：

| 模式 | 行为 |
|---|---|
| `交易/盘口/记录` | 跳转到交易记录卡片 |
| `MSI/季中赛/国际赛事` | 加载 MSI 赛程 |
| `看<队伍编码>` | 切换队伍（如 `看TES`） |
| `改<字段名> <值>` | 修改对应三维字段 / 追加笔记 |
| `TK <关键词>` | TK 搜索 |
| `<队伍>三维` | 选队伍 + 滚动到三维卡片 |
| 其他 | 按 TK 搜索兜底 |

快捷键：`Ctrl+K` 或 `/` 聚焦命令框，`Ctrl+S` 保存三维，`ESC` 关闭覆盖层。

#### 3. 三维画像（3D Data）

- 后端表 `team_3d_data`（队伍名、3个维度名+值、笔记、版本理解、更新时间）
- 前端显示为 3 列可编辑卡片（dim-1/dim-2/dim-3）
- 脏标记：任一维度的 `<input>` 或 `<textarea>` 变化触发 `markDirty()`
- 保存按钮变色 + 反馈提示
- 保存接口 `PUT /api/team-3d/{team}`

#### 4. TK 知识库

- 双源搜索：先调 RAG API（`localhost:8768`），再扫本地 TK 文件目录
- 搜索结果：每条显示 icon + 标题 + 预览（200字截断）+ 可展开全文
- 操作：展开/折叠 / 编辑 / 删除 / 新建
- TK 编辑器：模态覆盖层，支持修改 content / tags / team

#### 5. MSI 赛程

- 从 `/api/schedules` 按 `event=MSI` 过滤
- 表格显示：日期、时间、对阵、阶段/格式
- 区域切换：ALL / LPL / LCK / INTL 快速聚焦不同赛区的观察口径

#### 6. 交易记录（Trade Records）

- 多游戏支持：LOL / CS / 无畏契约 / 足球
- 记录结构：比赛名 / 时间 / 输赢预测 / 大小分预测 / 比分判断 / 理由 / 信心度 / 结算结果
- 操作：记录 / 结算（赢/输/走水）/ 删除
- **「沉淀TK」**：把一条交易记录转成 TK 知识条目
- **「从当前队伍带入」**：自动从当前选中队伍的赛程和三维评分推断倾向
- 统计面板：总记录 / 已结算 / 赢 / 输 / 走水 / 命中率

## 后端 API（main.py）

### 完整 API 列表

| 方法 | 路由 | 说明 |
|---|---|---|
| GET | `/api/teams` | 返回所有有 roster / 3D / INTL 标记的队伍 |
| GET | `/api/schedules` | 赛程查询（筛选：event / region / team / date / upcoming） |
| GET | `/api/players` | 查询队伍首发选手（team 参数必填） |
| GET | `/api/team-3d/{team}` | 查询队伍三维数据 |
| PUT | `/api/team-3d/{team}` | 更新队伍三维数据 |
| GET | `/api/tk/search` | TK 知识库搜索（q 必填，team/limit 可选） |
| POST | `/api/tk` | 新建 TK 条目（写文件 + 触发 RAG 重索引） |
| DELETE | `/api/tk/{filename}` | 删除 TK 条目 |
| GET | `/api/wiki/team/{team}` | 读取队伍 Wiki 页面（markdown → HTML） |
| GET | `/api/wiki/concept/{concept}` | 读取战术概念 Wiki 页面 |
| GET | `/api/profile-full/{team}` | 完整画像（Wiki 优先 → SKILL.md 回退） |
| GET | `/api/analyst/{team}` | LLM 分析师（中年电竞人 + 心语悦无言双分析） |
| GET | `/api/links` | 侧边栏外链接（TK图/日报/战力排行等） |
| GET | `/api/trades` | 交易记录列表（筛选：game / result） |
| POST | `/api/trades` | 新建交易记录 |
| PUT | `/api/trades/{id}` | 更新交易记录（结算等） |
| DELETE | `/api/trades/{id}` | 删除交易记录 |
| GET | `/api/trades/stats` | 交易统计汇总 |
| GET | `/` | 首页 SPA（返回 dist/index.html） |

### 后端关键依赖

- **数据库**：SQLite（`/home/ubuntu/lol_data/英雄联盟数据库.db`）
- **TK 目录**：`/home/ubuntu/workspace/knowledge/wiki/小雪电竞/原始资料/tk/`
- **RAG 服务**：`http://localhost:8768`（搜索 + 重索引）
- **LLM**：DeepSeek API（config.yaml 读取 key）
- **SKILL 目录**：`~/.hermes/skills/` + `~/.hermes/profiles/xiaotian/skills/`
