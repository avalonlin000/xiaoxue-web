# Vibe Coding Loop — 开发验证闭环

> 每次开新功能/改模块时，用这个提示词开启一轮 loop。

## 提示词（贴给AI）

```
【Vibe Coding Loop】

本轮模块：[模块名]

需求卡：
- 做什么：
- 数据从哪来：
- 长什么样：
- 我验证：

循环规则：
1. 实现后必须输出「改了哪些文件 + 我验证的具体步骤」
2. 我验证完给反馈：✅通过 / ❌[位置] [现象] [预期]
3. ❌时只修指出的问题，不趁机重构别的
4. ✅时更新 progress.md，等我开下一轮
5. 不动什么：[明确写出来]
```

## 使用示例

```
【Vibe Coding Loop】

本轮模块：TK面板排序优化

需求卡：
- 做什么：TK搜索结果按日期降序排列，最新在最上面
- 数据从哪来：RAG API /api/tk/search
- 长什么样：和现在一样，只是顺序对调
- 我验证：搜"版本理解"→ 第一条应该是今天的最新TK

循环规则：
1. 实现后必须输出「改了哪些文件 + 我验证的具体步骤」
2. 我验证完给反馈：✅通过 / ❌[位置] [现象] [预期]
3. ❌时只修指出的问题，不趁机重构别的
4. ✅时更新 progress.md，等我开下一轮
5. 不动什么：不改TK编辑器弹窗、不改画像栏
```

## 节奏

```
需求卡（你填）
    ↓
AI 实现 → 报告改动范围
    ↓
你验证 → ✅ 通过 → commit → 下一轮
         → ❌ 不通过 → 给修改工单 → 回到"AI实现"
```

## 关键规则

- **"不动什么"不写，AI就会顺手重构周边代码**——这是最常炸的地方
- commit 节点 AI 自己记在 progress.md，不用你提醒
- 一轮只做一个模块，跑通了再开下一个
