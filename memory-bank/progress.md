## 当前验收口径

| 模块 | 验收 |
|------|------|
| 运维 | `vibe-status` 服务全 active，proxy residue 为 none，根分区低于 85% 优先；85%-92% 警告，92% 以上必须清理 |
| 盘口草稿 | 前端新建、读取、删除只走 `/api/market-notes`，用于保存交易时原始判断；赛后复盘由钧钧和小雪在后台对话中完成，前端不承载复盘表单；不自动交易、不自动写 TK/画像 |
| 赛前交易判断日报 | 从当日赛程读取双方队伍，结构化读取 active `type=trading_note`，生成“基础面 → 命中队伍交易备注 → 市场分歧 → 交易小结 → BP待确认”；无备注或数据不足时写“暂不推荐”；“小雪记到 HLE：...”可写入队伍 TK，队伍不明确不落库 |
| LOL 日报 v2 | 生产只走 `daily_pipeline.py --publish`；`postprocess_daily_report.py`/`inject_pre_match_trading_layer.py` 为停用兼容文件，不得进入生产；发布前 readiness manifest 必须是当天完整刷新、`mode=full / ok=true` 且两个阶段成功 |
| BP 八步法 | `/api/lineup-workflow/prepare` 校验蓝红方和双方五位置，返回固定八步与 `保留/降权/取消/等待`；前端只整理输入，小雪对话执行动态判断 |
| 画像页 | 优先 Wiki，其次 SKILL.md，再用数据库只读兜底；兜底画像必须标明来源和“不替代人工画像” |
| 版本理解 | `/api/version-understanding/{team}` 只读聚合三维字段与 TK 条目；前端只展示，不自动生成新判断 |
| TK 搜索 | `/api/tk/search` 同时读 knowledge-rag 和 Wiki TK 文件；日期从 frontmatter/mtime 提取，结果按日期降序，不能按文件名字母序提前截断 |
| TK资料库 | 默认最新优先；今天/7天/30天/月/全部可切换；队伍与关键词可筛选；每次加载 30 条；点开读取完整正文且不显示来源/文件名等后台字段 |
| memory-bank | 新对话先读 `README.md`、改模块读 `modules.md`、汇报进度读本文件 |
| 模块隔离 | 默认运行 `scripts/acceptance_check.py` 做只读逐模块验收；某一模块失败后继续检查其余模块；架构门禁要求后端严格三层、跨模块只走 `public.py`，前端跨模块只走 `public.js`/语义事件；只有显式传入写入开关才允许做临场记录写入验收 |
